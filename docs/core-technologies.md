# 多坐标系下的闭合轨道渲染

## 一、背景：三套坐标系

卫星轨道数据从计算到屏幕渲染，需要经过三套坐标系的转换：

| 坐标系 | 原点 | 竖轴方向 | 特点 |
|--------|------|----------|------|
| **惯性系（ECI）** | 地心 | Z 轴朝北极 | 固定在恒星空间，不随地球自转，是轨道计算的原始输出 |
| **地固系（ECEF）** | 地心 | Z 轴朝北极 | 固定在地球表面，随地球一起旋转，用于表达卫星的地理位置 |
| **场景系（Scene）** | 地心 | Y 轴朝北极 | Three.js 渲染引擎的坐标约定，Y-Up，用于 GPU 渲染 |

其中惯性系与地固系之间的转换依赖**格林尼治恒星时（GMST）**——它反映了地球相对恒星空间的旋转角度，每时每刻都在变化。

## 二、问题：轨道在地固系中不闭合

系统需要在 3D 地球上绘制完整的轨道环——即卫星绕行一整圈（约 95 分钟）的轨迹，起点与终点应完全重合。

直觉做法是对每个时间点分别计算 ECI 位置，再用**各自时刻的 GMST** 转换到地固系。但这样做存在一个问题：在 95 分钟的轨道周期内，地球自转了约 24°。虽然轨道首尾在天上（惯性系）回到了同一位置，但地固系的参考框架已经旋转了 24°，导致首尾两个点在地固系中发生错位——轨道环被"撕开"了。

## 三、解决方案：固定 GMST 策略

核心思路：取**起始时刻的 GMST**，统一用于转换轨道上所有点。相关实现在 [satMath.ts:315-387](src/utils/satMath.ts#L315-L387) 的 `calculateCompleteOrbitPath` 函数中。

物理含义上，这等同于在惯性空间中"冻结"地球的自转状态。由于卫星轨道在惯性空间中本身是闭合的（开普勒椭圆），冻结参考框架后，轨道在地固系中也自然闭合。

代价方面，该轨道环仅对应起始时刻的地球朝向。随着模拟时间推进，需以新的时间点重新计算。实际实现中，这一重算由 React 的 `useMemo` 依赖自动触发，不会造成用户可感知的延迟。

## 四、地固系到场景系的坐标映射

闭合轨道计算完成后，还需经过一次坐标轴方向变换才能用于 Three.js 渲染。[satMath.ts:21-27](src/utils/satMath.ts#L21-L27) 的 `mapEcefToScene` 定义了这一映射关系：

| 方向 | 地固系 | 场景系 |
|------|--------|--------|
| 北极（朝上） | Z | Y |
| 90°E（朝右） | Y | X |
| 本初子午线（朝前） | X | Z |

这是一个与固定 GMST 策略无关的独立问题——无论轨道如何计算，地固系与 Three.js 场景系之间的轴方向差异始终存在，必须做这层转换。项目中所有涉及坐标的模块（卫星位置、轨道线、地面站标记、经纬度定位）统一使用同一套映射，确保画面中所有元素的位置一致。

## 五、地球纹理对齐

完成坐标映射后，还涉及一个容易被忽略的对齐问题：地球球面纹理的方向。

Three.js 默认的球体 UV 映射中，正前方（+X 方向）对应本初子午线。但经过上一节的坐标轴映射后，场景的正前方变成了 +Z 方向。因此在 [Earth3D.tsx:153](src/components/Earth3D.tsx#L153) 中，球体网格需要绕 Y 轴旋转 -90°，使纹理上的本初子午线对齐到场景的 +Z 方向。

这也与前述两个问题无关，纯粹是纹理坐标与场景坐标轴的对齐。但如果旋转角度存在误差，卫星图标将与底图产生肉眼可见的经度偏移，在国境线等地理参考物附近尤为明显。

## 六、同轨道面去重渲染

千帆星座等大型星座中，同一轨道面上部署了多颗卫星，它们共享同一条轨道轨迹。如果不做处理，同一圈轨道会被重复绘制多次。

在 [Earth3D.tsx:545-569](src/components/Earth3D.tsx#L545-L569) 中，系统利用轨道数据对象的引用作为唯一标识，通过 `Map` 进行去重。同一轨道面的卫星共享同一个轨道数据对象，渲染时只绘制一次。

## 七、数据流全景

```
TLE 轨道根数
      │
      ▼
  SGP4/SDP4 轨道传播  ──→  惯性系（ECI）位置
      │
      ▼
  固定 GMST 转换  ──→  地固系（ECEF）位置（轨道闭合）
      │
      ▼
  坐标轴映射  ──→  场景系（Scene）位置
      │
      ▼
  Three.js 渲染（BufferGeometry + LineBasicMaterial）
```
